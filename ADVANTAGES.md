# Преимущества Ifarma BE Study Planner

> Документ предназначен для экспертов жюри хакатона.
> Описывает функциональные и технические достижения продукта по пяти критериям оценки.

---

## 1️⃣ Проработка дизайна исследования и регуляторное соответствие

### Что реализовано

**Автоматический выбор дизайна** — система анализирует CVintra, T½ и предпочтения пользователя и
предлагает оптимальный дизайн из четырёх классов: `2×2`, `2×3×3`, `2×4`, `parallel`.

**Регуляторные проверки в реальном времени** — эндпоинт `/compliance` анализирует каждый параметр
исследования по трём нормативным документам:
- **Решение Совета ЕЭК №85** (ЕАЭС) — основной регуляторный стандарт для РФ;
- **EMA Guideline on the Investigation of Bioequivalence** (CHMP/EWP/QWP/1401/98 Rev. 1);
- **FDA Guidance for Industry: Bioequivalence Studies with Pharmacokinetic Endpoints**.

Каждый вывод системы сопровождается точной ссылкой на пункт нормативного документа, например:
> *«Wash-out 7 дней ≥ 5×T½ — соответствует требованиям. Решение №85, п.7.1; EMA BE Guideline §4.1.9»*

**RSABE** — система автоматически распознаёт высоковариабельные препараты (CV > 30%) и рекомендует
реплицированный дизайн с применением метода масштабирования (RSABE/ABEL), что напрямую соответствует
актуальным требованиям регуляторов.

**Синопсис протокола** — генерируемый документ содержит все разделы, требуемые стандартами CRO:
цели, задачи, дизайн, критерии включения/исключения, PK-параметры, статистическую методологию,
план мониторинга безопасности, биоаналитический метод, расчёт выборки, регуляторные замечания.

**Полный пакет документов** — эндпоинт `/protocol` генерирует:
- Полноценный протокол исследования;
- Информационный листок добровольца (ИЛД);
- Форму информированного согласия (ИС);
- Краткую брошюру исследователя.

## 2️⃣ Инженерная реализация и технологическая архитектура

### Технологический стек

| Уровень | Технологии |
|---------|-----------|
| Backend | Python 3.12, FastAPI, Pydantic v2 |
| AI/LLM | OpenAI GPT-4 (async-ready, fallback) |
| Export | python-docx (DOCX), fpdf2 + DejaVu Unicode (PDF) |
| Frontend | Vanilla JS, Chart.js 4, CSS3 animations |
| Deploy | Docker, Render.com (render.yaml) |

### Архитектурные решения

**Модульность** — каждый слой системы изолирован в отдельный пакет:
```
config.py          — константы (один источник правды)
models/domain.py   — Pydantic-схемы с валидацией
pk_data/source.py  — мини-БД PK-параметров
design/logic.py    — алгоритм выбора дизайна
stats/sample_size.py — TOST расчёт выборки
reg/checks.py      — регуляторные проверки
synopsis/          — шаблонная генерация синопсиса
llm/client.py      — LLM-клиент с fallback
export/formatter.py — экспорт DOCX/PDF
cases/library.py   — Case-Based Reasoning библиотека
api/main.py        — FastAPI-приложение (10 эндпоинтов)
```

**Воспроизводимость** — шаблонная генерация без LLM всегда даёт детерминированный результат
(один и тот же ввод → один и тот же синопсис). LLM используется опционально.

**Graceful degradation** — при отсутствии OPENAI_API_KEY система продолжает работать
в режиме шаблонной генерации (`LLM_FALLBACK_TO_TEMPLATE = True`).

**REST API** — 10 задокументированных эндпоинтов (OpenAPI docs на `/docs`):
- `POST /design` — расчёт и синопсис (шаблон)
- `POST /design-llm` — расчёт и синопсис (GPT-4)
- `POST /export/docx` — экспорт в DOCX
- `POST /export/pdf` — экспорт в PDF
- `GET  /cases` — библиотека прецедентов
- `POST /cases/search` — поиск похожих кейсов
- `POST /cases/save` — сохранение кейса
- `POST /protocol` — полный протокол + ИЛД + ИС + ИБ
- `POST /translate` — перевод синопсиса (GPT-4)
- `POST /compliance` — нормативная проверка

## 3️⃣ Пользовательский интерфейс и опыт

### Дизайн и UX

**3-шаговый мастер** — форма разбита на логические разделы: Препарат → Дизайн и вариабельность →
Популяция. Быстрый ввод через «чипы» (омепразол, метформин, амоксициллин, аторвастатин).

**Интерактивная визуализация выборки** — коллапсируемая панель с тремя ползунками:
- CVintra (10–70%)
- Мощность исследования (70–95%)
- Граница биоэквивалентности (70–90%)

График в реальном времени показывает зависимость N от CVintra, подсвечивает текущее значение.

**Экспорт одним кликом** — после генерации синопсиса доступны кнопки:
📄 Скачать DOCX · 📑 Скачать PDF · 🌐 Перевести на английский · 📋 Полный протокол

**Compliance-панель** — автоматически появляется после расчёта, показывает цветовую индикацию
соответствия нормативным требованиям (🟢 / 🟡 / 🔴) с прямыми ссылками на пункты документов.

**База прецедентов** — после каждого расчёта система показывает 1–3 похожих завершённых
исследования с дизайном, выборкой, результатами и регуляторными ссылками.

**Тёмная тема** — профессиональный тёмный интерфейс с фиолетово-голубой цветовой схемой,
нейросетевой анимацией на фоне, адаптивной вёрсткой (mobile-ready).

**Доступность** — поддержка `prefers-reduced-motion`, семантическая вёрстка.

## 4️⃣ Инновационный потенциал и оригинальность подхода

### Уникальные особенности

**Case-Based Reasoning (CBR)** — встроенная библиотека обезличенных прецедентов исследований
биоэквивалентности. При вводе нового МНН система автоматически находит наиболее похожие
завершённые проекты по алгоритму скоринга (совпадение МНН + близость CV + совпадение дизайна).
Позволяет опираться на реальный опыт при обосновании плана исследования.

**Мультиформатный экспорт с корпоративным шаблоном** — DOCX и PDF с фирменным заголовком,
метаданными, нижним колонтитулом. Готовый документ можно немедленно отправить в печать
или загрузить в СЭД компании.

**Регуляторная линза в реальном времени** — система не просто генерирует документ,
но и проверяет его разделы против актуальных версий нормативных актов с точными ссылками.
Это снижает риск замечаний при подаче документов в регуляторные органы.

**Автоматический перевод с сохранением терминологии** — перевод синопсиса на английский
адаптирует регуляторные ссылки (Решение №85 → EMA BE Guideline), что ускоряет подготовку
международных заявок.

**Полный документальный пакет** — на основе одного синопсиса система формирует 4 документа:
Протокол + ИЛД + ИС + Краткая брошюра исследователя. Исключает дублирование данных,
обеспечивает согласованность.

**Интерактивный калькулятор выборки** — позволяет в режиме диалога с заказчиком менять
параметры и сразу видеть, как изменится N. Инструмент для презентаций и переговоров.

## 5️⃣ Проработка бизнес-модели и план достижения MVP

### Целевой рынок

**Первичные пользователи:** фармацевтологи и менеджеры клинических исследований в фармкомпаниях
(объём рынка КИ в России — ~60 млрд руб./год).

**Вторичные пользователи:** CRO-организации, академические центры КИ, стартапы в регтехе.

### Бизнес-модель

| Сегмент | Модель | Цена |
|---------|--------|------|
| Крупные фармкомпании | SaaS, корпоративный тариф | 150–300 тыс. руб./мес |
| CRO | API-интеграция (per-call) | 500–1500 руб./протокол |
| Академические центры | Freemium (лимит 5 протоколов/мес) | Бесплатно / 15 тыс. руб./мес |

### Дорожная карта

**Месяц 1–2 (текущий MVP — прототип):**
- [x] Автовыбор дизайна, расчёт выборки, регуляторные проверки
- [x] Генерация синопсиса (шаблон + GPT-4)
- [x] Экспорт DOCX/PDF
- [x] Case-Based Reasoning библиотека
- [x] Визуализация выборки
- [x] Полный документальный пакет
- [x] Перевод + нормативная проверка

**Месяц 3–4 (MVP v1):**
- [ ] Личный кабинет с историей проектов
- [ ] Реальная БД МНН (подключение к ГРЛС/EMA EPAR)
- [ ] Интеграция с DocuSign/СЭДО для ИС
- [ ] Корпоративные шаблоны с логотипом заказчика

**Месяц 5–6 (MVP v2):**
- [ ] Совместная работа команды (real-time коллаборация)
- [ ] Версионирование протоколов и трек изменений
- [ ] Интеграция с eCTD-системами (регуляторные досье)
- [ ] Мобильное приложение для быстрого согласования

### Оценка затрат на разработку до MVP v1

|--------|-----------|
| Бэкенд-разработчик (Python/FastAPI), 3 мес | 450 тыс. руб. |
| Фронтенд-разработчик (React), 3 мес | 375 тыс. руб. |
| Медицинский писатель (консультант) | 150 тыс. руб. |
| Регуляторный консультант | 100 тыс. руб. |
| Инфраструктура (cloud, LLM API) | 75 тыс. руб. |
| **Итого** | **~1 150 тыс. руб.** |

### Ключевые риски и митигация

| Риск | Вероятность | Митигация |
|------|------------|-----------|
| Изменение нормативной базы (Решение №85 v2) | Средняя | Версионирование правил, подписка на обновления |
| Низкое качество LLM в сложных случаях | Средняя | Обязательная верификация специалистом, шаблонный fallback |
| Утечка данных клинических исследований | Низкая | Обезличивание в CBR, on-premise вариант деплоя |
| Конкуренция от крупных вендоров (Veeva, Medidata) | Средняя | Фокус на локальный регуляторный контекст (ЕАЭС) |
